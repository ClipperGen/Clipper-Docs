## Bug Details & Resolusi

### 1. Masalah CORS (Cross-Origin Request Blocked) pada Admin FE
* **Masalah**: Pada browser console, request ke API admin seperti `/api/v1/admin/system/health`, `/api/v1/admin/users`, dan `/api/v1/admin/audit-logs` diblokir oleh Same-Origin Policy karena memanggil `http://localhost:8000` bukannya domain API asli.
* **Penyebab**: Variabel lingkungan Next.js dengan prefix `NEXT_PUBLIC_` dievaluasi secara statis pada saat kompilasi (*build-time*). Karena `Clipper_Admin_FE/Dockerfile` tidak memiliki deklarasi `ARG NEXT_PUBLIC_API_URL` dan `docker-compose.yml` tidak meneruskannya sebagai argumen build untuk service `admin`, build Next.js fallback ke default `"http://localhost:8000/api/v1"`.
* **Resolusi**:
  1. **Fallback Hostname Dinamis (Sisi Klien)**: Mengubah inisialisasi variabel `API_BASE` pada file klien API agar secara otomatis mendeteksi jika aplikasi berjalan di browser dan nilainya adalah localhost namun diakses lewat domain publik. Klien akan memetakan domain admin ke domain API yang sesuai secara dinamis.
  2. **Konfigurasi Argumen Build Docker**: Menambahkan deklarasi argumen build ke Dockerfile dan mendefinisikannya di berkas docker-compose agar terkompilasi dengan benar saat deployment.

## Detil Perubahan Kode & File

### 1. Clipper_Admin_FE
* **[src/lib/api-client.ts](file:///home/ubuntu/projects/clipper/Clipper_Admin_FE/src/lib/api-client.ts)**:
  Mengubah inisialisasi `const API_BASE` menjadi variabel `let` dengan logika deteksi browser (`window.location.hostname`):
  ```typescript
  let API_BASE =
    process.env.NEXT_PUBLIC_API_URL || "http://localhost:8000/api/v1";

  if (typeof window !== "undefined") {
    const host = window.location.hostname;
    if (host !== "localhost" && host !== "127.0.0.1" && API_BASE.includes("localhost")) {
      if (host.includes("admin")) {
        const apiHost = host.replace("admin", "api");
        API_BASE = `${window.location.protocol}//${apiHost}/api/v1`;
      } else if (host.includes("clipper") && !host.includes("clipper-api")) {
        const apiHost = host.replace("clipper", "clipper-api");
        API_BASE = `${window.location.protocol}//${apiHost}/api/v1`;
      } else {
        const parts = host.split(".");
        if (parts.length > 2) {
          parts[0] = "clipper-api";
          API_BASE = `${window.location.protocol}//${parts.join(".")}/api/v1`;
        } else {
          API_BASE = `${window.location.origin}/api/v1`;
        }
      }
    }
  }
  ```
* **[Dockerfile](file:///home/ubuntu/projects/clipper/Clipper_Admin_FE/Dockerfile)**:
  Menambahkan `ARG NEXT_PUBLIC_API_URL` dan `ENV NEXT_PUBLIC_API_URL` pada stage `builder` dan stage `runner` agar Next.js dapat merekam variabel tersebut saat proses kompilasi bundel static.
