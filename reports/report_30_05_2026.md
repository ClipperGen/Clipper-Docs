# Laporan Integrasi Frontend Clipper-Gen: Pemangkasan API Penuh & Perbaikan Bug

## Ringkasan Eksekutif
Frontend Clipper-Gen (Next.js 16 + Clerk Auth) sebelumnya beroperasi dengan alur data yang rusak — data mock, ketidaksesuaian tipe dengan backend Go, dan beberapa error runtime yang mencegah fungsi nyata. Sesi ini melakukan pemangkasan full-stack: menyelaraskan semua antarmuka API frontend dengan struktur respons backend Go yang sebenarnya, menghapus data mock hardcode dari 6+ komponen, menambahkan endpoint unggah file nyata ke backend, dan memperbaiki dua error runtime kritis (loop re-render tak terbatas dan referensi fungsi undefined). Frontend sekarang memuat data pengguna, proyek, aktivitas, transaksi, dan notifikasi nyata dari API, dan unggahan file disimpan ke disk melalui endpoint multipart baru.

## Detail Bug & Resolusi

### 1. "Maximum update depth exceeded" — Loop Re-render Tak Terbatas (sidebar.tsx:215)
* **Masalah**: Error runtime React — `Maximum update depth exceeded` dipicu di komponen `SidebarContent` pada tooltip pencapaian. Selur dashboard tidak dapat digunakan karena re-render tak terbatas.
* **Penyebab**: Hook `useApi()` (`frontend/src/lib/api.ts:270`) membuat `new ClipperApi(getToken)` pada **setiap render**, menghasilkan referensi objek baru setiap kali. Ini diteruskan ke `useDashboardData()` yang memiliki `useEffect(() => { fetchAll(api) }, [api, fetchAll])` — referensi `api` yang berubah memicu efek pada setiap siklus render, yang memperbarui store, yang me-render ulang sidebar, yang membuat `api` baru, dan seterusnya.
* **Solusi**: Memoisasi instans `ClipperApi` dengan `useMemo`:
  ```typescript
  // Sebelum (rusak)
  export function useApi(): ClipperApi {
    const { getToken } = useAuth();
    return new ClipperApi(getToken);
  }

  // Sesudah (diperbaiki)
  export function useApi(): ClipperApi {
    const { getToken } = useAuth();
    return useMemo(() => new ClipperApi(getToken), [getToken]);
  }
  ```

### 2. "addProject is not defined" — ReferenceError Runtime (project/page.tsx, editor/page.tsx)
* **Masalah**: `ReferenceError: addProject is not defined` menjebak halaman Project dan placeholder Editor saat merender `FileUploadDialog`.
* **Penyebab**: `simulateUpload()` di `file-upload-dialog.tsx:209` memanggil `addProject({...})` — fungsi yang tidak pernah diimpor atau didefinisikan dalam cakupan komponen. Ini adalah sisa dari pola mock yang dihapus di mana `addProject` seharusnya berasal dari store tetapi telah diganti oleh `createProject`.
* **Solusi**: Mengganti seluruh mock `simulateUpload` dengan fungsi `handleUpload` nyata yang memanggil `api.uploadProject(file)` untuk setiap file, lalu menyegarkan daftar proyek melalui `fetchProjects(api)`.

### 3. "socket hang up" / Frontend 500 errors di belakang Cloudflare tunnel
* **Masalah**: Next.js 16 middleware di belakang Cloudflare tunnel menghasilkan error `socket hang up` karena header `x-middleware-rewrite` yang sudah usang menunjuk ke `localhost`.
* **Penyebab**: Next.js 16 mendepresiasi konvensi `middleware.ts` untuk mode "proxy". Middleware menulis ulang ke `localhost`, yang rusak saat diakses melalui Cloudflare tunnel.
* **Solusi**: Menerapkan solusi admin — middleware menghapus header `x-middleware-rewrite` yang menunjuk ke `localhost` sebelum meneruskan respons.

### 4. Data Tidak Pernah Memuat — Gerbang Autentikasi Chicken-and-Egg
* **Masalah**: Dashboard menampilkan state kosong secara permanen — tidak ada data yang pernah dimuat dari API.
* **Penyebab**: `useDashboardData()` mengunci semua panggilan fetch pada `isLoggedIn`, yang `false` secara default dan tidak pernah diatur oleh alur autentikasi Clerk. Hook tidak pernah memanggil `fetchAll(api)`.
* **Solusi**: Menghapus gerbang `isLoggedIn`. Dengan Clerk, autentikasi ditangani oleh middleware — hook memanggil `fetchAll(api)` secara langsung.

## Masalah yang Diketahui & Mitigasi
* **Pipeline Pemrosesan Video**: Unggahan file disimpan ke disk dan catatan proyek dibuat, tetapi pemrosesan video nyata (FFmpeg/Whisper/Celery) belum dipicu untuk file yang diunggah — hanya pencernaan URL YouTube yang memicu pipeline penuh. **Mitigasi**: Menghubungkan jalur unggah file ke antrean pemrosesan Celery.
* **Submodule Frontend Detached HEAD**: Commit submodule frontend dibuat dalam keadaan detached HEAD. **Mitigasi**: Berhasil didorong ke `master` melalui `HEAD:refs/heads/master`, tetapi seharusnya berada di branch yang tepat untuk pekerjaan mendatang.
* **Direktori `uploads/`**: File disimpan ke direktori `uploads/` lokal di direktori kerja backend Go. Tidak cocok untuk produksi — membutuhkan penyimpanan objek (S3/R2). **Mitigasi**: Dilacak untuk migrasi infrastruktur.
* **Biner Terkompilasi di Repo**: `go-backend/api` (biner ELF terkompilasi) ditampilkan sebagai modifikasi di git. Harus ditambahkan ke `.gitignore`.
