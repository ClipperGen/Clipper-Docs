## Bug Details & Resolusi

### 1. "Stuck on Ingesting" / "Stuck on Analyzing" (UPDATED)

* **The Problem**: Project video stuck on INGESTING or ANALYZING status, tidak pernah selesai. User report: "its stuck on ingestion and that just failed". Meskipun video original (dQw4w9WgXcQ) seharusnya memiliki 99 markers, di production tetap fallback.

* **The Cause**: 
  - **Primary**: YouTube bot detection di production server. HTTP requests dari server IP tertentu mungkin diblokir oleh YouTube.
  - **Secondary**: Beberapa video YouTube TIDAK MEMILIKI "Most Replayed" markers data (e.g., 6wysozn4e28, xvFZjo5PgG0)
  - Video dQw4w9WgXcQ (Rick Astley Original): return 99 markers ✓ (di local)
  - Sistem sudah handle dengan fallback ke `_pick_fallback_anchors()`

* **The Solusi**: 
  - ✅ **NEW**: Tambah yt-dlp fallback di `fetch_most_replayed_heatmap()`
  - yt-dlp sering bisa bypass bot detection lebih baik dari HTTP requests
  - Heatmap function sekarang: HTTP requests (6 UAs) → yt-dlp fallback → predefined fallback
  - Fallback logic sudah implement di `process_video_ingest` (lines 199-202)
  - User need understand: Not all videos have heatmap data (ini expected YouTube behavior)

### 2. CORS Errors (Non-Critical)
