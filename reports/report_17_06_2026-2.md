# Laporan Hapus Klip: Endpoint DELETE /clips/:id

## Ringkasan Eksekutif
Aksi "Hapus" pada dropdown menu Perpustakaan Klip sebelumnya hanya menampilkan toast stub tanpa integrasi backend. Sesi ini menambahkan endpoint `DELETE /api/v1/clips/:id` ke backend Go Fiber dengan verifikasi kepemilikan, menghubungkan klien API dan hook React Query, serta mengganti tombol hapus dengan dialog konfirmasi `AlertDialog` yang aman.

## Fase Selesai / Fitur Terimplementasi

### Fase 1: Backend Go — Handler DeleteClip
* **Detail Implementasi**: Menambahkan fungsi `DeleteClip(c *fiber.Ctx) error` ke `handlers/clips.go`. Handler mem-parse parameter UUID `:id`, memverifikasi kepemilikan melalui JOIN `generated_shorts` → `project_videos` → `projects` dengan filter `p.user_id = ?`, menghapus baris `generated_shorts`, dan mengembalikan `204 No Content`. Jika klip tidak ditemukan atau bukan milik pengguna, mengembalikan `404 Not Found`.
* **Integritas Data**: Kepemilikan ditegakkan melalui query COUNT sebelum delete — tidak ada pengguna yang bisa menghapus klip pengguna lain. Query delete hanya mempengaruhi satu baris (`WHERE id = ?`).
* **Optimasi**: Satu query COUNT untuk verifikasi + satu query DELETE. Tidak ada JOIN tambahan yang tidak perlu.

### Fase 2: Backend Go — Pendaftaran Rute
* **Detail Implementasi**: Menambahkan `protected.Delete("/clips/:id", handlers.DeleteClip)` ke `cmd/api/main.go` di grup rute terlindung, tepat setelah rute `GET /clips`. CORS sudah mengizinkan method `DELETE`.
* **Integritas Data**: Rute memerlukan `AuthMiddleware` — permintaan tanpa autentikasi ditolak sebelum mencapai handler.

### Fase 3: Frontend — Klien API
* **Detail Implementasi**: Menambahkan metode `deleteClip(id: string)` ke objek `api` di `api-client.ts`. Menggunakan `request<void>(/clips/${id}, { method: "DELETE" })`. Mengembalikan promise tanpa body respons (204).
* **Integritas Data**: Tipe `void` memastikan tidak ada parsing JSON yang tidak perlu. Error HTTP dilempar sebagai exception untuk ditangani oleh React Query.

### Fase 4: Frontend — Hook React Query
* **Detail Implementasi**: Menambahkan hook `useDeleteClip()` ke `use-api.ts`. Menggunakan `useMutation` dengan `mutationFn` yang memanggil `api.deleteClip(id)`. `onSuccess` memanggil `invalidateQueries({ queryKey: ["clips"] })` untuk menyegarkan daftar klip secara otomatis.
* **Integritas Data**: Invalidasi kueri memicu pengambilan ulang data dari server — UI selalu menampilkan data terkini.

### Fase 5: Frontend — Dialog Konfirmasi
* **Detail Implementasi**: Mengganti `toast.info('Clip deleted')` stub dengan `AlertDialog` dari Radix UI. Menambahkan state `deleteClipId` untuk melacak klip mana yang akan dihapus. Dropdown menu item memanggil `setDeleteClipId(clip.id)` alih-alih toast langsung. `AlertDialog` menampilkan judul "Delete clip?", deskripsi "This action cannot be undone", tombol Cancel, dan tombol Delete (merah). Tombol Delete memanggil `deleteClip.mutate()` dengan `onSuccess` (toast sukses + reset state) dan `onError` (toast error dengan pesan).
* **Integritas Data**: Dialog hanya muncul saat `deleteClipId` tidak null. `onOpenChange` mereset state saat dialog ditutup. Mutasi hanya dipanggil saat tombol Delete diklik, bukan saat dialog dibuka.

## Masalah yang Diketahui & Mitigasi

* **File Fisik Tidak Dihapus**: Delete hanya menghapus baris database. File MP4 di `exports/renders/` tidak dihapus. **Mitigasi**: Akan diintegrasikan saat migrasi ke object storage (S3/R2) — item #3 di TODO.
* **Tanpa Decrement clips_count**: Hapus klip tidak memperbarui `projects.clips_count`. **Mitigasi**: Menambahkan pemicu atau sinkronisasi penghitung tingkat aplikasi.
* **Tanpa Optimistic Update**: UI menunggu respons server sebelum memperbarui. **Mitigasi**: Tambahkan optimistic update untuk pengalaman lebih cepat jika diperlukan.
