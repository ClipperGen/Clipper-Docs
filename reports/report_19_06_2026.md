# Laporan Integrasi Pipeline YouTube Import → Video Processing

## Ringkasan Eksekutif
Sesi ini menyelesaikan seluruh jalur integrasi dari YouTube URL import hingga video processing pipeline. Tiga masalah kritis diperbaiki: React error #185 (infinite re-render loop), YouTube metadata fetch yang terblokir bot-detection, dan pipeline async yang belum terkoneksi. Hasil akhirnya adalah sistem end-to-end yang memungkinkan user memasukkan URL YouTube, project dibuat dengan metadata lengkap, dan worker secara otomatis memulai proses download → transkripsi → analisis.

## Bug Details & Resolusi

### 1. React Error #185 — Infinite Re-render Loop di Dashboard

* **The Problem**: Dashboard mengalami `Uncaught Error: Maximum update depth exceeded` (React #185). Browser freeze, console membanjiri error.

* **The Cause**: `useSyncApiData()` di `dashboard/layout.tsx` menggunakan `const store = useAppStore()` tanpa selector. Ini berarti komponen berlangganan ke seluruh Zustand state. Tujuh `useEffect` hook dalam komponen semuanya memiliki `store` sebagai dependency. Setiap efek yang memanggil `store.login()` atau `store.setProjects()` memicu perubahan state → new `store` ref → semua efek re-run → infinite loop.

* **The Solusi**:
  - Mengganti semua setter dari `store.xxx()` menjadi `useAppStore.getState().xxx()` — referensi stabil yang tidak re-render.
  - Mengganti semua reader dari `store.xxx` menjadi selector (`useAppStore(s => s.xxx)`).
  - Menambahkan ref guards (`if (loadedRef.current) return`) untuk mencegah re-trigger.
  - File: `Clipper_FE/src/hooks/use-sync-api.ts`

### 2. YouTube Metadata Fetch Gagal — Bot Detection

* **The Problem**: yt-dlp gagal mengambil metadata YouTube dari VPS. Error `Sign in to confirm you're not a bot` di semua player client (`web_creator`, `android`, `ios`, `mweb`). Dampak: `POST /api/v1/projects` mengembalikan 502.

* **The Cause**: YouTube mendeteksi request dari IP datacenter VPS sebagai bot. yt-dlp menggunakan player clients yang memicu challenge. `--cookies-from-browser firefox` juga tidak berfungsi karena tidak ada Firefox di Docker container.

* **The Solusi**:
  - Mengganti metadata fetch dari yt-dlp ke **YouTube oEmbed API** (`/youtube/v1/oembed`). Endpoint ini bebas biaya, tidak memerlukan autentikasi, dan tidak terpengaruh bot detection.
  - Fallback ke yt-dlp hanya untuk durasi (best-effort), dengan default 600 detik (10 menit) jika gagal.
  - File: `Clipper_BE/apps/go-backend/internal/utils/youtube.go`

### 3. Frontend Import URL Tidak Terkoneksi ke Backend

* **The Problem**: Tombol "Import URL" di frontend hanya membuat project fake secara lokal. Tidak ada panggilan ke API.

* **The Cause**: `handleNewProject` di project-grid.tsx membuat object `{ id, title, thumbnail, status }` secara hardcoded. Tidak menggunakan `api.createProject()`.

* **The Solusi**:
  - Menghubungkan `handleNewProject` ke `api.createProject({ source_url })` di api-client.ts.
  - Menggunakan `useCreateProject` React Query hook untuk invalidation cache.
  - File: `Clipper_FE/src/components/dashboard/project-grid.tsx`, `Clipper_FE/src/lib/api-client.ts`

### 4. API Tidak Push Task ke Worker

* **The Problem**: Setelah project YouTube dibuat, tidak ada task yang dikirim ke Celery worker. Worker tidak pernah memproses video.

* **The Cause**: `CreateProject` handler di Go API tidak meng-push `process_video_ingest` task ke Redis queue.

* **The Solusi**:
  - Menambahkan `celery.SendTask("process_video_ingest", ...)` ke `CreateProject` handler.
  - Status project diatur ke `INGESTING` sebelum push task.
  - File: `Clipper_BE/apps/go-backend/internal/handlers/project.go`

### 5. Project Delete Tidak Berfungsi

* **The Problem**: Tidak ada endpoint DELETE untuk project. Tombol delete tidak ada di frontend.

* **The Solusi**:
  - Menambahkan `DELETE /projects/:project_id` handler dengan cascading delete (shorts → videos → project).
  - Menambahkan `useDeleteProject` hook di frontend.
  - Menambahkan konfirmasi dialog sebelum delete.
  - File: `Clipper_BE/apps/go-backend/cmd/api/main.go`, `Clipper_BE/apps/go-backend/internal/handlers/project.go`, `Clipper_FE/src/hooks/use-api.ts`

### 6. Status Case Mismatch — DB vs Frontend

* **The Problem**: Database menyimpan status sebagai `FAILED` (uppercase), tetapi frontend mencari `failed` (lowercase). Project yang gagal tidak menampilkan apapun di grid (karena `statusConfig.draft` undefined).

* **The Solusi**:
  - Menambahkan `.toLowerCase()` di sync hook saat mapping status.
  - Menambahkan `?? statusConfig.draft` fallback di project-grid.tsx untuk status yang tidak dikenali.
  - File: `Clipper_FE/src/hooks/use-sync-api.ts`, `Clipper_FE/src/components/dashboard/project-grid.tsx`

## Known Issues & Mitigasi

* **YouTube Bot Detection pada Download**: YouTube memblokir video download dari IP datacenter. oEmbed metadata selalu berhasil, tetapi download video asli terkadang terkena bot-check dari VPS. Mitigasi: worker mencoba beberapa player client secara berurutan (`web_creator → android → ios → mweb`), dan beberapa video berhasil diunduh. Solusi jangka panjang: user dapat menyediakan `cookies.txt` dari browser mereka, atau menggunakan YouTube Data API v3.

* **Durasi Video Default**: oEmbed tidak menyediakan durasi video. Sistem menggunakan default 600 detik (10 menit). Durasi aktual hanya tersedia setelah yt-dlp berhasil download. Mitigasi: durasi dapat diperbarui secara manual atau setelah video berhasil diproses.

* **Transcription Pipeline**: Task `process_video_analysis` memerlukan client-submitted transcription (bisa dari Whisper, Deepgram, dll). Saat ini worker hanya mengunduh video dan mengekstrak audio, lalu menunggu transcription dari client. Mitigasi: integrasi Whisper di worker untuk transkripsi otomatis.
