# Laporan API Perpustakaan Klip Clipper Gen: Agregasi Shorts Lintas Proyek

## Ringkasan Eksekutif
Halaman Perpustakaan Klip sebelumnya menampilkan 8 klip mock hardcode dengan nol integrasi backend. Sesi ini menambahkan endpoint baru `GET /clips` ke backend Go Fiber yang mengagregasi semua `generated_shorts` di seluruh proyek pengguna menjadi satu perpustakaan yang dapat dikueri, dan menghubungkan komponen frontend `ClipsLibrary` untuk mengonsumsi data API nyata dengan status pemuatan, penanganan error, dan state kosong.

## Fase Selesai / Fitur Terimplementasi

### Fase 1: Backend Go â€” Endpoint Klip Baru
* **Detail Implementasi**: Membuat `Clipper_BE/apps/go-backend/internal/handlers/clips.go` dengan handler `ListAllClips`. Menggunakan JOIN SQL mentah melalui GORM untuk mengagregasi `generated_shorts` â†’ `project_videos` â†’ `projects`, difilter berdasarkan ID pengguna yang terotentikasi. Mendukung paginasi (`limit`/`offset`), pengurutan (`newest`/`viral`/`duration`), dan penyaringan status (`LOCKED`/`UNLOCKED`/`RENDERING`/`COMPLETED`). Mengembalikan header `X-Total-Count` untuk kesadaran paginasi klien.
* **Integritas Data**: Kepemilikan ditegakkan melalui filter `p.user_id = ?` pada JOIN. Tidak ada kebocoran data lintas pengguna yang mungkin. Filter status dipetakan langsung ke kolom `generated_shorts.export_status`.
* **Optimasi**: Satu kueri agregasi alih-alih N+1 kueri per proyek. Subkueri untuk `viral_score` hanya saat `clip_id` TIDAK NULL. Limit default 50, maks 200.

### Fase 2: Backend Go â€” Pendaftaran Rute
* **Detail Implementasi**: Menambahkan `protected.Get("/clips", handlers.ListAllClips)` ke `cmd/api/main.go` di grup rute terlindung, bersamaan dengan endpoint yang ada. Endpoint memerlukan `AuthMiddleware` â€” permintaan tanpa autentikasi ditolak sebelum mencapai handler.
* **Integritas Data**: Rute ditempatkan sebelum `/projects` untuk hierarki URL yang bersih. Tidak ada pembatasan khusus admin â€” pengguna hanya melihat klip mereka sendiri.

### Fase 3: Frontend â€” Klien API + Hook
* **Detail Implementasi**: Menambahkan antarmuka `ApiClip` ke `api-client.ts` yang cocok dengan field struct respons Go (tag JSON snake_case). Menambahkan fungsi `getClips(params?)` dengan pembuatan kueri parameter opsional. Menambahkan hook React Query `useClips(params?)` ke `use-api.ts` dengan kunci kueri `["clips", params]` untuk invalidasi cache otomatis saat parameter berubah.
* **Integritas Data**: Parameter kueri opsional dengan default yang masuk akal (limit=50, sort_by=newest). Hook mengembalikan data `undefined` saat pemuatan, ditangani oleh komponen.

### Fase 4: Frontend â€” Pemangkasan Ulang ClipsLibrary
* **Detail Implementasi**: Penulisan ulang lengkap `clips-library.tsx`. Menghapus semua 8 entri `mockClips` hardcode (120 baris data mock). Menambahkan panggilan hook `useClips()` dengan parameter live. Menambahkan transformer `mapApiClipToItem()` untuk mengonversi snake_case API ke camelCase frontend. Menambahkan 3 state UI baru: pemuatan (8 kartu skeleton), kosong (pesan terpusat dengan panduan generate), error (ikon + pesan). Menghapus lencana `platform` dan metadata `style` dari kartu grid (tidak tersedia di backend). Mempertahankan semua fitur yang ada: toggle tampilan grup/daftar, filter pill (all/published/draft), dropdown pengurutan, bilah statistik, menu dropdown aksi.
* **Integritas Data**: `views`, `likes`, `shares` bernilai default 0 â€” baris statistik tersembunyi saat semua nol. `viralScore` dirender dari data backend nyata melalui subkueri tabel `clips`. `duration` dihitung dari `endTime - startTime` di backend.
* **Optimasi**: Pemuatan skeleton alih-alih spinner. State error dengan petunjuk coba ulang. State kosong memandu pengguna untuk menghasilkan shorts dari proyek.

## Masalah yang Diketahui & Mitigasi

* **Metrik Sosial Hilang**: `views`, `likes`, `shares` selalu mengembalikan 0. Tidak ada pipeline pemasukan analitik yang ada. **Mitigasi**: Baris statistik tersembunyi saat semua nol. Menambahkan endpoint analitik saat integrasi platform sosial dibangun.
* **Platform/Gaya Tidak Tersedia**: Backend tidak memiliki kolom `platform` atau `style` pada `generated_shorts`. Ini murni kosmetik di data mock. **Mitigasi**: Dihapus dari UI. Menambahkan kolom ke model saat target ekspor diimplementasikan.
* **Hapus Belum Terhubung**: Aksi "Hapus" dropdown masih menampilkan toast stub. Membutuhkan endpoint `DELETE /clips/:id`. **Mitigasi**: Implementasikan saat persyaratan manajemen klip ditetapkan.
* **Tanpa Pembaruan Jumlah Klip Per Proyek**: Pembuatan/penghapusan klip tidak memperbarui `projects.clips_count`. **Mitigasi**: Menambahkan pemicu atau sinkronisasi penghitung tingkat aplikasi.
