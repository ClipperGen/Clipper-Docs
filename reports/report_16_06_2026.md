## Bug Details & Resolusi

### 1. Go Build Failure: Unused Import (MODERATE)
* **The Problem (Masalah)**: `go build ./cmd/api/` failed with `"os" imported and not used` in `main.go`.
* **The Cause (Penyebab)**: The `os` package was imported but only used indirectly through `config.GetEnv()`. The direct import was dead code.
* **The Solving (Solusi)**: Removed the unused `"os"` import from `cmd/api/main.go:6`.

### 2. Admin FE Docker Build: Missing Standalone Output (MODERATE)
* **The Problem (Masalah)**: Docker build failed with `"/app/.next/standalone": not found` during the runner stage.
* **The Cause (Penyebab)**: `Clipper_Admin_FE/next.config.ts` did not set `output: "standalone"`, so Next.js didn't generate the standalone build output required for Docker.
* **The Solving (Solusi)**: Added `output: "standalone"` to `next.config.ts`.

### 3. File Upload Dialog Type Error (MINOR)
* **The Problem (Masalah)**: TypeScript error `Type 'UploadFile' is missing properties from type 'File'` in `file-upload-dialog.tsx`.
* **The Cause (Penyebab)**: Passed the wrapper `UploadFile` object instead of the underlying `File` object to `uploadVideo.mutateAsync()`.
* **The Solving (Solusi)**: Changed `{ file, title }` to `{ file: file.file, title }` to pass the actual `File` instance.

## Known Issues & Mitigasi

* **File Upload Not Triggering Processing**: File uploads via `POST /projects/upload` create records but the Python worker doesn't process them yet — only YouTube URLs trigger the `process_video_ingest` pipeline. **Mitigation**: Connect upload path to Celery processing queue.
* **Clips Library Still Mocked**: The `ClipsLibrary` component still uses local `mockClips` data. The backend doesn't have a dedicated "all clips across all projects" endpoint yet. **Mitigation**: Add `GET /clips` endpoint to Go API, or aggregate from projects.
* **Settings Page Not Fully Wired**: Referral stats and team data are fetched via React Query hooks but the Settings page components still use local state. **Mitigation**: Update Settings page to consume store data from API sync.
* **No Real-time Processing Updates**: Editor page doesn't poll for processing status. Clips show static data until page refresh. **Mitigation**: Add polling or WebSocket for live status updates.
* **Caddy Config Assumed**: The unified compose mounts `Clipper_FE/Caddyfile.prod` — if the file changes format, the frontend proxy breaks. **Mitigation**: Add Caddy config validation to CI.
