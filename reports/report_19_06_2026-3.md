# Laporan Arsitektur Client-Side WASM Processing & Auto-Generate Clips

## Ringkasan Eksekutif
Sesi ini melengkapi transformasi arsitektur Clipper Gen dari server-side video processing ke client-side WASM processing. Server kini HANYA mengambil metadata YouTube (title, duration, thumbnail) dan heatmap (Most Replayed moments) tanpa mendownload video penuh. Client (browser) menangani transcription via Whisper WASM dan video processing via ffmpeg.wasm. Clip otomatis digenerate dari heatmap YouTube dengan fallback ke predefined segments. Hasil: eliminasi total YouTube bot detection issues, nol penyimpanan video di server, skalabilitas tak terbatas.

## Bug Details & Resolusi

### 1. Clip Tidak Ter-generate untuk YouTube URL

* **The Problem**: `process_video_ingest` untuk YouTube URLs (tanpa file upload) hanya set status ke AWAITING_TRANSCRIPTION tanpa membuat GeneratedShort records. User tidak melihat clip apapun di library.

* **The Cause**: Logika asli hanya skip video download untuk YouTube URLs, tetapi tidak implement auto-generation dari heatmap. Sistem menunggu transcription dari client terlebih dahulu.

* **The Solusi**:
  - Memodifikasi `process_video_ingest` untuk YouTube-only projects:
    - Fetch heatmap via `fetch_most_replayed_heatmap(video_id)`
    - Gunakan `_pick_heatmap_anchors()` untuk memilih peak moments
    - Fallback ke `_pick_fallback_anchors()` jika heatmap tidak tersedia
    - Buat GeneratedShort records dengan start_time/end_time
    - Set project status ke COMPLETED
  - File: `Clipper_BE/apps/processing-engine/app/workers/tasks.py`

### 2. YouTube Thumbnail 400 Errors

* **The Problem**: Next.js image optimizer (/next/image) gagal mem-proxy thumbnail YouTube (i.ytimg.com) dengan error 400. Gambar thumbnail tidak tampil.

* **The Cause**: Domain i.ytimg.com dan ytimg.com tidak terdaftar di `images.remotePatterns` next.config.ts. Next.js image optimizer menolak permintaan ke domain eksternal yang tidak terdaftar.

* **The Solusi**:
  - Tambahkan remotePatterns untuk YouTube domains:
    ```typescript
    images: {
      remotePatterns: [
        { protocol: 'https', hostname: 'i.ytimg.com' },
        { protocol: 'https', hostname: 'ytimg.com' },
      ],
    }
    ```
  - File: `Clipper_FE/next.config.ts`

### 3. CORS Errors - Cloudflare Insights Beacon

* **The Problem**: Console error: "CORS request did not succeed" untuk static.cloudflareinsights.com/beacon.min.js. Status code: (null).

* **The Cause**: Cloudflare Analytics script di-inject oleh Cloudflare proxy, tetapi request gagal karena CORS restriction.

* **The Solusi**:
  - Nonaktifkan Cloudflare Analytics di Next.js config:
    ```typescript
    experimental: {
      disableCloudflareAnalytics: true,
    }
    ```
  - Tambahkan CSP headers via middleware untuk security hardening
  - File: `Clipper_FE/next.config.ts`, `Clipper_FE/src/middleware.ts`

### 4. Package @gerganov/whisper-wasm Not Found

* **The Problem**: Docker build frontend gagal dengan error: "GET https://registry.npmjs.org/@gerganov%2fwhisper-wasm - 404". Package tidak tersedia di npm.

* **The Cause**: Package name salah. Package yang benar adalah `whisper-wasm` (tanpa @gerganov/ prefix).

* **The Solusi**:
  - Ganti package name di package.json:
    ```json
    "whisper-wasm": "^0.0.1"
    ```
  - Update import di use-whisper.ts:
    ```typescript
    import { createModel, TranscriptionResult } from 'whisper-wasm';
    ```
  - File: `Clipper_FE/package.json`, `Clipper_FE/src/hooks/use-whisper.ts`

### 5. Clip Playback - Toast Only (No Actual Playback)

* **The Problem**: Klik tombol play di clip card hanya menampilkan toast "Playing clip preview..." tanpa memutar video.

* **The Cause**: Belum terintegrasi dengan YouTubeEmbed component.

* **The Solusi**:
  - Buat `YouTubeEmbed` component dengan support start/end time
  - Tambahkan modal dialog di ClipsLibrary
  - Ganti toast action dengan modal + YouTubeEmbed
  - Extract video_id dari video_url menggunakan utility function
  - File: `Clipper_FE/src/components/shared/youtube-embed.tsx`, `Clipper_FE/src/components/dashboard/clips-library.tsx`, `Clipper_FE/src/lib/youtube-utils.ts`

## Known Issues & Mitigasi

* **CSP Blocking API Calls**: CSP di middleware terlalu restriktif dan memblokir requests ke clipper-api.glassgallery.my.id. Mitigasi: CSP sudah diupdate untuk allow API domain. Perlu clear cache Cloudflare dan browser.

* **Cloudflare Insights Beacon**: Masih muncul warning di console meskipun disableCloudflareAnalytics: true. Mitigasi: CSP yang baru allow static.cloudflareinsights.com. Jika masih error, perlu check Cloudflare Tunnel configuration.

* **Font Preload Warnings**: Warning "preload not used within a few seconds" untuk woff2 fonts. Mitigasi: Sudah ditambahkan display: 'swap' di Geist font config. Warning ini non-critical dan hanya cosmetic.

* **favicon.ico 404**: favicon.ico tidak ada. Mitigasi: Tambahkan favicon file di public/ directory.

* **Heatmap Unavailable for Some Videos**: Beberapa video YouTube tidak memiliki "Most Replayed" data. Mitigasi: Fallback ke predefined segments (_pick_fallback_anchors) sudah diimplementasikan.

## Arsitektur Baru - Flow Lengkap

```
User Submit YouTube URL
    ↓
Go API: Validasi + Buat ProjectVideo record
    ↓
Celery Task: process_video_ingest
    ↓
    [YouTube-only path - NO DOWNLOAD]
    ↓
    Fetch Metadata (title, duration, thumbnail) via yt-dlp
    ↓
    Fetch Heatmap via fetch_most_replayed_heatmap()
    ↓
    Generate Anchors: heatmap peaks OR fallback segments
    ↓
    Buat GeneratedShort records (start_time, end_time)
    ↓
    Set Project Status: COMPLETED
    ↓
Frontend: Display clips di ClipsLibrary
    ↓
User click PLAY → Modal dengan YouTubeEmbed
    ↓
YouTubeEmbed: iframe dengan video_id + ?start=X&end=Y
    ↓
Video berputar di YouTube player (tidak re-hosted)
    ↓
(OPTIONAL) User transcribe via Whisper WASM
    ↓
(OPTIONAL) User refine clips via ffmpeg.wasm
```
